第一章 概论

git工作流程

  • 开局一张图

git工作区、暂存区、版本库

  • 工作区

    • 就是你在电脑里能看到的目录。

  • 暂存区

    • 英文叫stage, 或index。一般存放在 “.git目录下” 下的index文件（.git/index）中，所以我们把暂存区有时也叫作索引（index）。

  • 版本库

    • 工作区有一个隐藏目录.git，这个不算工作区，而是Git的版本库。

    • 三者关系

git基本操作

  • git中文件的三种状态
    • 已提交(committed):已提交表示数据已经安全的提交到本地数据库中。
    • 已修改(modified):已经修改了文件，但是还没有保存到数据库中。
    • 已暂存(staged):对一个已经修改的文件做了当前版本的标记，使其包含在下次提交的快照中。
  • git项目的三个工作区域
    • git仓库：保存项目的元数据与对象数据库的地方。当从计算机拷贝仓库时，拷贝的数据就是这些。
    • 工作目录：是对项目的某个版本独立提取出来的内容，这些从git仓库的压缩的文件中提取出来的文件，放在磁盘上供使用者使用与修改。
    • 暂存区域：暂存区域作为一个文件，保存下次将要提交的文件列表信息。一般放在git的仓库目录中。
  • git工作的基本流程
    • 在工作目录中修改文件。
    • 将文件的快照放入暂存区域保存。
    • 提交更新，也就是提交暂存区域的文件的新的内容。

  • 配置用户信息(配置基本信息，仅需配置一次，可以通过 git config –list 查看 )

    • ```
      $ git config –global user.name “John Doe”
      $ git config –global user.email johndoe@example.com

      git config core.editor Code //绑定VScode

      git help
      git –help
      man git

      git init //本地初始化一个仓库，创建子目录.git，包含仓库中所有的必要文件。
      git add .< sources file> //添加需要跟踪的文件
      $ git add LICENSE
      $ git commit -m ‘initial project version’ //提交

      get clone https://github.com/RockShirely/rockshirely.github.io local_dir

      git status <-s> //需要跟踪新的文件，通过git add new_file_name 实现

      git diff

      git commit
      git commit -a //会把所有跟踪的文件提交

      git rm file_name

      git log -p //每次提交内容的差异
      git log ….//详细查看手册

      git commit –amend

      git clone url //克隆远程仓库到本地目录
      git remote //列举指定的每一个远程仓库
      git remote -v //列举每一个指定的远程仓库

    • 使用--global选项，该命令只需要运行一次，无论在该系统中做任何事情，git都会使用这些信息。

    • 要使用不同的用户进行操作，可以在相应的项目目录下面利用--global进行配置。

第二章 Git基础

2.1 获取Git仓库

  • 获取git仓库的方式有二：
    • 在现有的项目或者目录下导入所有文件到git中。
    • 从一个服务器克隆现有的git仓库。

2.1.1 在现有的目录中初始化仓库

  • 首先作的就是进入相应的目录，然后执行以下的git命令。

    • ```bash
      $ git init

2.1.2 克隆现有的仓库

  • git的克隆是克隆的是该git仓库上几乎所有的数据。

  • 执行git clone命令时，默认配置下的远程仓库下的每一个文件的每一个版本都被克隆下来。

  • 指令命令格式：

    • ```bash
      git clone [url] [本地仓库名]

    • 上述的信息表名工作目录很干净，所有跟踪的文件未被修改。同时也表明当前目录的文件没有出现未被跟踪的新文件。

  • 当存在新文件时，使用git status命令再查看时，会出现如下的信息.以README为例。

    • ```bash
      $ echo ‘my project’ >README
      $ git status
      $ On branch master No commits yet Untracked files: (use “git add …” to include in what will be committed) README nothing added to commit but untracked files present (use “git add” to track)

    • 上述的信息表示目前的文件readme处于被跟踪状态，也就是处于暂存状态。信息changes to be committed表示当前文件处于暂存状态。

    • git add的参数可以是具体的文件，也可以是相应的目录，如果是目录，表示该目录所有的文件纳入跟踪的对象范围。

2.1.6 暂存已修改的文件

  • 修改一个已经跟踪的文件，然后使用git status查看相应的信息，会发现如下：（以readme为例）

    • ```bash
      cohen@XIAOGE-PC2:/mnt/e/Alearning/git-learning-pro$ git status On branch master No commits yet Changes to be committed: (use “git rm –cached …” to unstage) new file: README Changes not staged for commit: (use “git add …” to update what will be committed) (use “git restore …” to discard changes in working directory) modified: README cohen@XIAOGE-PC2:/mnt/e/Alearning/git-learning-pro$

    • 对于上述文件状态的解读：

      • ??：表示新添加未被跟踪的文件。
      • A：表示新添加到暂存区域中的文件。
      • M：表示修改过的文件。
      • MM：第一个M表示已经放入暂存区域的需改，第二个M表示新的一次修改。

2.1.8 忽略文件

  • 当我们想将一些不需要跟踪的文件纳入忽略文件里面，那么可以创建一个名为.gitignore的文件，列出要忽略的文件模式即可。

  • ```bash
    $ cat .gitignore
    *.[oa]
    *~

  • 如果需要删除的文件已经修改过并且放到了暂存区域，那么通过git rm -f来实现强制删除。

  • 如果想把文件从git仓库中删除，但仍希望保流在当前工作目录中，即是说想额昂磁盘保留，但是git不再继续跟踪。可以使用--cached选项实现。

    • ```bash
      $ git rm –cached …

2.1.13 查看提交历史

  • git log查看日志。

2.2 撤销操作

  • 如果提交过程发现有相应的文件未提交，可以运行带有--amend选项的提交命令尝试重新提交。

    • ```bash
      $ git commit –amend

2.1.2 撤销对文件的修改

  • ```bash
    $ git checkout – …

    • 上述pb代替整个url简单操作。

2.3.3 从远程仓库拉取和查看

  • 从远程仓库获取数据，可以使用:

    • ```
      git fetch [remote-name]

    • 下列将master分支推送到origin服务器使用命令如下：

      • ```
        git push origin master

    • 上述命令会列出远程仓库的URL和分支信息。

2.3.6 远程仓库的移除与重命名

  • 通过命令git remote rename 去修改一个远程仓库的简写名。比如修改名为pbde 远程仓库:

    • ```bash
      git remote rename pb paul

2.4 打标签

  • git可以给历史中的某个提交打上标签，以表示重要。

2.4.1 列出标签

  • 在git中列出已有的标签使用命令：

    • ```bash
      git tag
      v0.3
      v1.0

  • 可以通过git show来查看标签信息和对应的提交信息

    • ```
      git show v1.4
      tag v1.4
      Tagger: Ben Straub ben@straub.cc
      Date: Sat May 3 20:19:12 2014 -0700
      my version 1.4
      commit ca82a6dff817ec66f44342007202690a93763949
      Author: Scott Chacon schacon@gee-mail.com
      Date: Mon Mar 17 21:52:11 2008 -0700
      changed the version number

    • 使用git show也看不到其他信息

    • $ git show v1.4-lw
commit ca82a6dff817ec66f44342007202690a93763949
Author: Scott Chacon <schacon@gee-mail.com>
Date: Mon Mar 17 21:52:11 2008 -0700
